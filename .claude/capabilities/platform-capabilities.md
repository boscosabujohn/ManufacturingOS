# Platform Capabilities

Comprehensive documentation of the platform's core competencies organized by domain. Use when discussing platform features, capabilities, user journeys, module specifications, or when designing new features that integrate with the platform.

## Use When

1. Discussing platform features and capabilities
2. Understanding user journeys
3. Reviewing module specifications
4. Designing new features that integrate with the platform
5. Creating feature documentation

## Core Modules

- **Core**: Masters (Customer, Vendor, Item, UOM)
- **Sales**: RFP, Quotes, Orders, CRM
- **Production**: BOM, Work Orders, Shop Floor
- **Procurement**: PO, RFQ, Goods Receipt
- **Inventory**: Stock, Warehousing, Transfers
- **Finance**: GL, Invoices, Payments, AR/AP
- **HR**: Employees, Payroll, Leave, Attendance
- **Quality**: Inspections, NCR, CAPA
- **Workflow**: Approvals, Routing, Notifications

## Integration Points

- REST APIs for all modules
- Event-driven architecture with message queues
- Webhook support for external integrations
- Multi-tenant architecture support
